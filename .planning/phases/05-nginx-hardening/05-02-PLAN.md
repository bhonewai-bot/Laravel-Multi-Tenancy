  <action>
Add gzip directives to the server block in both dev and prod configs. Place them after the security headers section (added by Plan 05-01) and before the location blocks.

### Gzip block (identical in both configs, per D-11, D-12, D-13, D-14):

```
    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_min_length 256;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;
```

Note: `text/html` is always compressed by nginx by default and does not need to be listed in `gzip_types`.

### Placement rules:
- In dev config: add after the 4 security header lines, before the `location ^~ /.well-known/` block
- In prod config port 80 block: add after the 4 security header lines, before the `location ^~ /.well-known/` block
- In prod config port 443 block: add after the 4 security header lines, before the `location ~ \.php$` block

Do NOT put gzip in the custom nginx.conf -- it belongs in each server block where it can be configured per-environment.
  </action>

<task type="auto">
  <name>Task 2: Add static asset caching and no-cache for PHP responses</name>
  <files>docker/nginx/conf.d/app.conf, docker/nginx/conf.d/prod/app.conf</files>
  <read_first>

  <action>
Add cache-aware location blocks for static assets and update the PHP location block with no-cache headers. CRITICAL: Every location block with its own `add_header` must ALSO repeat the security headers from Plan 05-01, because nginx `add_header` in a location overrides all server-level headers.

### Step 1: Add Vite-hashed asset location (per D-15)

Add this location block BEFORE the PHP location block in both configs:

```
    # Vite-hashed static assets (JS/CSS) — 1-year cache, immutable
    location ~* \.(js|css)$ {
        add_header Cache-Control "public, max-age=31536000, immutable";
        add_header X-Content-Type-Options "nosniff" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    }
```

Security headers X-Content-Type-Options and Referrer-Policy are repeated because this location block has its own `add_header`. CSP and HSTS are omitted from static asset locations because they are not relevant to JS/CSS file loads (the HTML page that loads them carries those headers).

### Step 2: Add non-hashed static asset location (per D-16)

Add this location block AFTER the Vite-hashed asset block and BEFORE the PHP location block:

```
    # Non-hashed static assets (images, fonts) — 1-day cache
    location ~* \.(jpg|jpeg|png|gif|ico|svg|webp|woff|woff2|ttf|eot)$ {
        add_header Cache-Control "public, max-age=86400";
        add_header X-Content-Type-Options "nosniff" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    }
```

### Step 3: Update PHP location block with no-cache and security headers (per D-17)

The PHP location block currently has NO `add_header` directives in either config. After this change, it will have `add_header` directives, so ALL security headers must be added to prevent server-level headers from being overridden.

In both dev and prod configs, add these lines inside the `location ~ \.php$` block (after the fastcgi_param lines):

```
        # No caching for dynamic responses
        add_header Cache-Control "no-cache, no-store, must-revalidate" always;
        # Security headers (must repeat because add_header in location overrides server-level)
        add_header Content-Security-Policy-Report-Only "REPORT_ONLY_VALUE" always;
        add_header Strict-Transport-Security "max-age=300; includeSubDomains" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

Where `REPORT_ONLY_VALUE` is:
- Dev: `"script-src 'self' 'unsafe-inline' 'unsafe-eval'; connect-src 'self' ws: wss:;"`
- Prod HTTP (port 80): `"script-src 'self' 'unsafe-inline' 'unsafe-eval'; connect-src 'self';"`
- Prod HTTPS (port 443): `"script-src 'self' 'unsafe-inline' 'unsafe-eval'; connect-src 'self';"`

### Step 4: Add security headers to the catch-all location block

The `location /` block (try_files) also needs security headers if it serves requests that don't match other blocks. Add:

```
        add_header X-Content-Type-Options "nosniff" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

In dev config, also add CSP-Report-Only and HSTS. In prod configs, add the same with the strict CSP.

### Location block ordering (both configs):

1. `location ^~ /.well-known/cf-custom-hostname-challenge/` (unchanged)
2. `location ~* \.(js|css)$` (NEW - Vite-hashed assets)
3. `location ~* \.(jpg|jpeg|png|gif|ico|svg|webp|woff|woff2|ttf|eot)$` (NEW - non-hashed static)
4. `location ~ \.php$` (MODIFIED - add Cache-Control + security headers)
5. `location /` (MODIFIED - add security headers)

The `.php$` regex location must come AFTER the static asset regex locations so that `.js` and `.css` files are served with cache headers, not proxied to PHP-FPM.
