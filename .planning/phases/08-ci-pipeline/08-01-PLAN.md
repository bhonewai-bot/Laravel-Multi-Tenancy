    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: "8.3"

      - name: Check code style
        run: vendor/bin/pint --test

Add a new `docker` job to `.github/workflows/ci.yml` after the `audit` job:

```yaml
  docker:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Build Docker image
        run: docker build .

      - name: Validate prod compose
        run: docker compose -f docker-compose.prod.yml config
```

Key points:
- `docker build .` validates the Dockerfile builds cleanly
- `docker compose config` validates the prod compose file is syntactically correct
- No need to start containers — just validate build/config succeeds
- Docker buildx is available by default on ubuntu-latest
- The docker compose config step will show env var warnings (MDB_PASSWORD etc.) but those are warnings, not errors — the step still passes
  </action>
