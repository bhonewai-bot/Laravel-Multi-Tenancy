<objective>
Create OPcache production configuration and mount it into the prod container.

Purpose: Enable OPcache bytecode caching and PHP 8.3 tracing JIT for 30-50% throughput improvement.

Output: New opcache.ini file and volume mount in docker-compose.prod.yml.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create OPcache ini file</name>
  <files>docker/php/conf.d/opcache.ini</files>
  <read_first>
    .planning/phases/07-opcache-performance/07-CONTEXT.md
    docker-compose.prod.yml
  </read_first>
  <action>
Create directory `docker/php/conf.d/` if it doesn't exist.

Create `docker/php/conf.d/opcache.ini` with these exact contents:

```ini
[opcache]
opcache.enable=1
opcache.validate_timestamps=0
opcache.max_accelerated_files=10000
opcache.memory_consumption=128
opcache.jit=1255
opcache.jit_buffer_size=128M
```

Explanation:
- `opcache.enable=1` — explicit enable (enabled by default, but explicit is clearer)
- `validate_timestamps=0` — no file stat checks in production (D-01)
- `max_accelerated_files=10000` — sufficient for Laravel (D-02)
- `memory_consumption=128` — 128MB OPcache storage (D-03)
- `opcache.jit=1255` — PHP 8.3 tracing JIT (D-04)
- `opcache.jit_buffer_size=128M` — 128MB JIT buffer (D-05)
  </action>
  <verify>
    <automated>grep -c 'opcache.jit=1255' docker/php/conf.d/opcache.ini && grep -c 'validate_timestamps=0' docker/php/conf.d/opcache.ini</automated>
  </verify>
  <acceptance_criteria>

  </acceptance_criteria>
  <done>
OPcache ini file created with all production settings including JIT tracing configuration.
  </done>
</task>

<task type="auto">
  <name>Task 2: Mount opcache.ini in prod compose</name>
  <files>docker-compose.prod.yml</files>
  <read_first>
    docker-compose.prod.yml
    docker/php/conf.d/opcache.ini
  </read_first>
  <action>
Add a volume mount to the `app` service in `docker-compose.prod.yml`:

`- ./docker/php/conf.d/opcache.ini:/usr/local/etc/php/conf.d/opcache.ini:ro`

Add this after the existing `env_file` line or in the volumes section if one exists. If the app service has no `volumes` key, add one.

Also add the same mount to the `queue` service (same image, same OPcache behavior).

Do NOT add to the `scheduler` service — it runs `schedule:work` which spawns short-lived artisan processes, OPcache is less beneficial there. (Optional: add if you want consistency.)

Do NOT modify `docker-compose.yml` (dev) — dev should keep default OPcache behavior with validate_timestamps=1.
  </action>
  <verify>
    <automated>grep -c 'opcache.ini' docker-compose.prod.yml</automated>
  </verify>
  <acceptance_criteria>

    - Mount path is /usr/local/etc/php/conf.d/opcache.ini:ro
    - docker-compose.yml (dev) is NOT modified
  </acceptance_criteria>
  <done>
OPcache ini mounted into app and queue services in prod compose. Dev compose untouched.
  </done>
</task>
