<objective>
Update both Docker Compose files to remove host bind-mounts from production app/queue services and fix all Dockerfile references from `DockerFile` to `Dockerfile`.

Purpose: The production compose file currently bind-mounts `./:/var/www` on app and queue services, which overwrites the baked-in code with the host filesystem — defeating the multi-stage build. Both compose files also reference `DockerFile` (capital F) which must be updated to match the renamed `Dockerfile`.

Output: Updated docker-compose.prod.yml (no app/queue bind-mounts, Dockerfile reference fixed) and docker-compose.yml (Dockerfile reference fixed).
</objective>
